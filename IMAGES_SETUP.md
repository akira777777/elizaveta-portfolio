# Настройка изображений для портфолио

## Проблема
Изображения проекта EFEST не загружаются, так как файлы отсутствуют в папке `assets/images/portfolio`.

## Решение

### Вариант 1: Добавить реальные изображения (рекомендуется)

1. Создайте папку (уже создана):
   ```
   assets/images/portfolio/
   ```

2. Добавьте изображения с правильными именами:
   - `freepik__subtle-variation-of-the-previous-electronic-music-__38650.jpg` (и варианты -sm.jpg, -md.jpg, .webp)
   - `freepik__upload__96337.jpg` (и варианты -sm.jpg, -md.jpg, .webp)
   - `freepik__upload__96338.jpg` (и варианты -sm.jpg, -md.jpg, .webp)

3. Для оптимизации создайте варианты:
   - `-sm.jpg` - маленькие (400px ширина)
   - `-md.jpg` - средние (800px ширина)
   - `.webp` - WebP версии для лучшей производительности

### Вариант 2: Использовать placeholder сервис

Временно можно использовать сервис placeholder.com, изменив пути в HTML на:
```
https://via.placeholder.com/896x1280/f0f0f0/999999?text=EFEST+Collection
```

### Вариант 3: Использовать существующие скрипты

В проекте есть скрипты для работы с изображениями:
- `create-placeholders.ps1` - создает placeholder'ы (уже запущен)
- `optimize-portfolio.ps1` - оптимизирует изображения
- `create-variants.ps1` - создает варианты размеров

## Текущее состояние

- ✅ Папка `assets/images/portfolio` создана
- ✅ Placeholder'ы созданы (SVG формат)
- ✅ Обработка ошибок добавлена в HTML (fallback на data URI)
- ⚠️ Реальные изображения отсутствуют

## Следующие шаги

1. Добавьте реальные изображения EFEST в папку `assets/images/portfolio/`
2. Или используйте скрипт `optimize-portfolio-images.ps1` для обработки ваших изображений
3. После добавления изображений сайт будет работать полностью
